## 1. 概要・目的

### 1.1 画面概要

SCR-003「対話トレーニング画面」は、学習者がAIと英語で自由に会話練習を行うための画面である。  
テキスト入力・音声入力の両方に対応し、AIからのレスポンスもテキスト・音声の両方で受け取ることができる。

### 1.2 目的

| # | 目的 |
|---|------|
| 1 | 実際の会話に近い形でIT英語の実践練習を提供する |
| 2 | 音声入力（STT）・音声出力（TTS）により、リスニング・スピーキングを強化する |
| 3 | AIがリアルタイムでフィードバックを返すことで、学習効率を高める |
| 4 | 会話履歴を保存し、学習進捗の振り返りを可能にする |

### 1.3 利用AWSサービス

| サービス | 用途 |
|----------|------|
| Amazon Transcribe | 音声 → テキスト変換（STT） |
| Amazon Bedrock (Claude 3.5 Sonnet) | AI応答生成・文法フィードバック |
| Amazon Polly | テキスト → 音声変換（TTS） |
| Amazon API Gateway | REST APIエンドポイント |
| AWS Lambda | バックエンドロジック実行 |
| Amazon DynamoDB | 会話履歴・セッション管理 |
| Amazon SQS | フィードバック生成の非同期処理 |
| Amazon S3 | 音声ファイル一時保存 |
| AWS Secrets Manager | APIキー・認証情報管理 |

## 2. 前提条件・依存関係

### 2.1 前提条件

- SCR-001（ログイン画面）・SCR-002（ホーム画面）の実装が完了していること
- AWS環境（dev / stg / prod）が構築済みであること
- Cognito ユーザープールが設定済みで、JWTトークン認証が機能していること
- フロントエンドのプロジェクト初期設定（ESLint / Prettier / TypeScript）が完了していること

### 2.2 依存するAPI・Lambda関数

| 依存先 | 内容 |
|--------|------|
| POST /sessions | 会話セッション開始API |
| POST /sessions/{sessionId}/messages | メッセージ送信API |
| GET /sessions/{sessionId}/messages | 会話履歴取得API |
| POST /transcribe | 音声→テキスト変換API |
| POST /tts | テキスト→音声変換API |
| SQS FeedbackQueue | フィードバック非同期生成キュー |

### 2.3 開発ルール（遵守事項）

- TypeScript使用・`any`型禁止
- 変数・関数名は `camelCase`、環境変数は `SCREAMING_SNAKE_CASE`
- APIキー・シークレットは必ず AWS Secrets Manager 経由で取得
- Bedrock呼び出しはExponential Backoffで最大3回リトライ
- フィードバック生成は必ずSQS経由の非同期処理（同期呼び出し禁止）
- Bedrockはクロスリージョン呼び出し（us-east-1）。リージョンをハードコーディングしない
- テストカバレッジ目標：全体80%以上、Bedrock/Transcribe連携Lambda関数は90%以上

```

   → MediaRecorder.start()
   → UIを「録音中」状態に変更

2. ボタンを離す

4. テキスト取得後、入力フィールドにセット
   → 自動で POST /messages を呼び出し
```

## 7. バックエンド実装詳細

### 7.1 MessageFn（メインLambda）処理フロー

```
1. リクエスト受信
   - JWT認証（API Gateway Authorizer）
   - sessionId・contentのバリデーション

2. DynamoDBにユーザーメッセージを保存

3. DynamoDBから会話履歴（直近20件）を取得

4. Bedrock (Claude 3.5 Sonnet) 呼び出し
   - システムプロンプト：IT英語会話アシスタントとして応答
   - 会話履歴をコンテキストとして渡す
   - Exponential Backoffで最大3回リトライ

5. AI応答をDynamoDBに保存

6. SQS FeedbackQueueにメッセージを投入
   - payload: { sessionId, messageId, userContent }

7. AI応答をフロントエンドへ返却
```

### 7.2 TranscribeFn 処理フロー

```
1. 音声BlobをS3に一時保存
   - バケット: {ENV}-it-english-audio-temp
   - キー: {userId}/{timestamp}.webm

2. Amazon Transcribeを起動
   - 言語コード: en-US
   - 完了までポーリング（最大30秒）

3. テキスト結果をフロントエンドへ返却

4. S3の一時ファイルを削除
```

### 7.3 FeedbackFn 処理フロー

```
1. SQSメッセージ受信
   - { sessionId, messageId, userContent }

2. Bedrock (Claude 3.5 Sonnet) でフィードバック生成
   - プロンプト: 文法・表現の誤りを指摘し、改善例を提示
   - Exponential Backoffで最大3回リトライ

3. DynamoDBの Messagesテーブルに role=feedback で保存

4. フロントエンドは次回の Polling時に取得
```

### 7.4 Bedrockプロンプト設計方針

| 用途 | システムプロンプトの要点 |
|------|--------------------|
| AI応答生成 | IT英語会話アシスタントとして応答。自然な会話を維持し、必要に応じて技術用語を使用 |
| フィードバック生成 | ユーザーの発言の文法・語彙・表現を分析。誤りの指摘と改善例を簡潔に提示 |

## 8. AWSサービス設定

### 8.1 Amazon Transcribe

| 設定項目 | 値 |
|----------|---|
| 言語 | `en-US` |
| 出力形式 | JSON（トランスクリプトのみ取得） |

| 音声形式 | WebM / WAV |

| 設定項目 | 値 |
|----------|---|

| リージョン | `us-east-1`（クロスリージョン呼び出し） |
| `max_tokens` | 1024 |
| `temperature` | 0.7（AI応答）/ 0.3（フィードバック） |
| リトライ | Exponential Backoffで最大3回 |
| 初回待機時間 | 1秒、次回2秒、次回4秒 |

### 8.3 Amazon Polly

| 設定項目 | 値 |
|----------|---|
| 音声 | `Joanna`（女性・英語） |
| エンジン | `neural` |
| 出力形式 | `mp3` |
| 返却方式 | Base64エンコードしてフロントエンドへ返却 |

### 8.4 Amazon SQS

| 設定項目 | 値 |
|----------|---|
| キュー名 | `{ENV}-FeedbackQueue` |
| キュータイプ | Standard Queue |
| メッセージ保持期間 | 4日 |
| 表示遅延 | 0秒 |
| 最大受信数 | 3（超過後Dead Letter Queueへ移動） |
| Dead Letter Queue | `{ENV}-FeedbackDLQ` |

### 8.5 IAMロール設定

| Lambda関数 | 必要な権限 |
|------------|----------|
| `MessageFn` | `dynamodb:PutItem`, `dynamodb:Query`, `bedrock:InvokeModel`, `sqs:SendMessage` |
| `TranscribeFn` | `s3:PutObject`, `s3:DeleteObject`, `transcribe:StartTranscriptionJob`, `transcribe:GetTranscriptionJob` |
| `TtsFn` | `polly:SynthesizeSpeech` |

## 9. API仕様

### 9.1 POST /sessions

**概要:** 会話セッションを開始する

**リクエストヘッダー:**
```
Authorization: Bearer {JWT}
Content-Type: application/json
```

**レスポンス 200:**
```json
{
  "sessionId": "uuid-xxxx",
  "status": "active",
  "createdAt": "2026-06-24T10:00:00Z"
}
```

---

### 9.2 POST /sessions/{sessionId}/messages

**概要:** メッセージを送信し、AI応答を受け取る

**リクエストボディ:**
```json
{
  "content": "I'd like to practice talking about cloud architecture."
}
```

**レスポンス 200:**
```json
{
  "messageId": "uuid-yyyy",
  "role": "assistant",
  "content": "Great! Let's start with the basics of cloud architecture...",
  "createdAt": "2026-06-24T10:01:00Z"
}
```

---

### 9.3 GET /sessions/{sessionId}/messages

**概要:** 会話履歴（メッセージ・フィードバック含む）を取得する

**クエリパラメータ:**
```
limit: 最大件数（デフォルト: 50）
lastKey: ページネーション用
```

**レスポンス 200:**
```json
{
  "messages": [
    { "messageId": "...", "role": "user", "content": "...", "createdAt": "..." },
    { "messageId": "...", "role": "assistant", "content": "...", "createdAt": "..." },
    { "messageId": "...", "role": "feedback", "content": "...", "createdAt": "..." }
  ],
  "lastKey": "..."
}
```

## 10. エラーハンドリング方針

### 10.1 フロントエンドのエラーハンドリング

| エラーケース | ユーザーへの表示 | リカバリー |
|----------|------------|----------|
| セッション開始失敗 | 「セッションを開始できませんでした」 + 再試行ボタン | 再試行ボタンでPOST /sessions再実行 |
| メッセージ送信失敗 | 「送信に失敗しました。再度お試しください」 | 入力内容を保持したまま再送信可能 |
| STT変換失敗 | 「音声を認識できませんでした」 | テキスト入力にフォールバック |
| TTS変換失敗 | テキストのみ表示（音声ボタンをグレイアウト） | テキスト表示は維持 |
| フィードバック取得失敗 | サイレントにスキップ | 次回のPollingで再取得 |
| ネットワーク切断 | 「接続が切断されました」 + 再接続ボタン | 再接続後にセッションを復元 |

### 10.2 バックエンドのエラーハンドリング

| エラーケース | 処理 |
|----------|------|
| Bedrockレートリミット | Exponential Backoffで最大3回リトライ。超過後503返却 |
| Transcribeタイムアウト | 30秒超過で408返却。S3ファイルは必ず削除 |
| DynamoDB書き込み失敗 | リトライなしで500返却。CloudWatchにエラーログ出力 |
| SQS投入失敗 | メッセージ送信自体は成功として返却。フィードバックはスキップ |
| FeedbackFn失敗 | DLQへ移動。最大3回再試行後に廃棄 |

## 11. テスト計画

### 11.1 ユニットテスト

#### フロントエンド

| テスト対象 | テスト内容 |
|----------|----------|
| `useSession` | セッション開始・終了の正常・異常ケース |
| `useChat` | メッセージ送信・受信・エラーハンドリング |
| `useAudioRecorder` | 録音開始・停止・Blob生成 |
| `useFeedbackPolling` | Polling間隔・フィードバック取得・クリーンアップ |
| `MessageBubble` | role別の表示スタイル・コンテンツ表示 |
| `MicButton` | 押下・離すイベント・状態遷移 |

#### バックエンド

| テスト対象 | テスト内容 |
|----------|----------|
| `MessageFn` | 正常フロー・Bedrockエラー・DynamoDBエラー・JWT認証失敗 |
| `TranscribeFn` | S3保存・Transcribe成功・タイムアウト・ファイル削除 |
| `TtsFn` | Polly成功・Pollyエラー |
| `FeedbackFn` | SQS受信・Bedrockフィードバック・DynamoDB保存 |
| Exponential Backoff | リトライ回数・待機時間の検証 |

### 11.2 結合テスト

| テストシナリオ | 検証内容 |
|------------|----------|
| テキスト入力からAI応答までのエンドツーエンド | 入力 → API → Bedrock → 応答表示 |
| 音声入力からAI応答までのエンドツーエンド | 録音 → STT → メッセージ送信 → AI応答 |
| TTS再生フロー | AI応答 → POST /tts → 音声再生 |
| フィードバック非同期フロー | メッセージ送信 → SQS → FeedbackFn → Pollingで取得 |
| セッション終了フロー | 終了ボタン → セッションステータス更新 → ホーム画面遷移 |

### 11.3 カバレッジ目標

|------|------|
| フロントエンド全体 | 80%以上 |
| `MessageFn` | 90%以上 |
| `TranscribeFn` | 90%以上 |
| `FeedbackFn` | 90%以上 |
| `TtsFn` | 80%以上 |
