# IT業界特化型・音声対話英会話トレーニングアプリ

## 1. プロジェクト概要

- **アプリ名:** IT-English Trainee (AWS Edition)

- **コンセプト:** IT現場（特にインド人エンジニアとの協業）で発生するリアルな業務シーンを、AWSのAIサービスを活用して音声でロールプレイ学習できるアプリ。

- **ターゲット:** 日本人エンジニア、PM、ディレクター。

- **主要技術スタック:** AWS (Amazon Bedrock, Amazon Transcribe, Amazon Polly) を活用した音声対話基盤。

## 2. 業務フローと学習シナリオ

以下の5つの「あるある」シーンを基本シナリオとして実装する。

1. **【朝会】進捗報告とブロック:** 予期せぬバグによる遅延報告と、今日の予定調整。

2. **【朝会】勤怠連絡:** 通院や私用による中抜け・早退の連絡と引き継ぎ。

3. **【業務中】設計相談:** DB設計のA案・B案の比較、パフォーマンス観点での意見交換。

4. **【業務中】バグ調査:** 再現しない不具合の調査依頼、テスト手順とログ共有の要請。

5. **【終業前】残業・デプロイ確認:** 作業遅延による残業申告と、夜間デプロイの影響確認。

## 3. 主要機能要件

### 3.1 音声ロールプレイ機能

- **音声入力 (STT):** Amazon Transcribeを利用し、ユーザーの発話をリアルタイムでテキスト化する。

- **対話生成 (LLM):** Amazon Bedrock (Claude 3.5 Sonnet推奨) を使用。AIが「インド人エンジニア」などの役割になりきり、文脈に沿った返答を生成する。

- **音声出力 (TTS):** Amazon Pollyを利用し、AIの返答を自然な英語音声で再生する。

### 3.2 フィードバック・評価機能

- **添削:** 会話終了後、ユーザーの英語表現に対して「よりIT現場で自然な言い回し」「誤解を与えない表現」をAIが提案する。

- **評価スコア:** 文法、流暢さ、IT用語の適切な使用の観点からスコア化する。

### 3.3 学習管理機能

- **履歴保存:** 過去の対話テキストとフィードバック内容を保存する。

- **進捗ダッシュボード:** 学習回数や苦手なシチュエーションを可視化する。

## 4. 画面遷移定義

1. **ホーム画面:** 今日の推奨シナリオと学習ステータスの表示。

2. **シナリオ選択画面:** 5つの基本シナリオおよび難易度の選択。

3. **対話トレーニング画面:** AIとのチャット/音声対話UI。発話ボタン、テキストログ、翻訳表示切替。

4. **フィードバック画面:** 添削結果、重要フレーズ解説、評価スコアの表示。

5. **学習履歴一覧画面:** 過去のトレーニング結果のリスト表示。

## 5. データモデル案 (Entity)

- **User:** ID, 名前, 英語レベル, 学習目標

- **Scenario:** ID, タイトル, 説明文, 初期プロンプト(Role), 難易度

- **ChatLog:** ID, UserID, ScenarioID, 発話内容(User/AI), 音声URL, タイムスタンプ

- **Feedback:** ID, ChatLogID, 総合スコア, 添削詳細テキスト

## 6. 非機能要件・インフラ

- **認証:** AWS Cognitoによるユーザー認証。

- **ストレージ:** 音声データはS3に保存。

- **API:** AWS Lambdaを介したサーバーレス構成。

- **拡張性:** 将来的には自社専用のプロジェクト用語辞書を追加できる設計とする。
